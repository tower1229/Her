# Timeline 北极星设计目标

> 状态：当前最高设计目标
> 目的：把 Timeline 项目的最终产品目标、系统边界、验收标准和设计原则固定下来，作为后续实现与文档的最高依据
> 读者：项目维护者、OpenClaw 集成开发者、persona / selfie / 其他下游 skill 的设计者

## 1. 一句话定义

安装 `timeline-plugin` 后，OpenClaw 在普通涉及“当下”与“回忆”的对话中，应先基于会话与 memory 检索既定事实；若事实缺失，则结合 persona 与真实世界时间进行人格化生成；并将结果写入统一 canon，使后续聊天和自拍等下游 skill 都共享同一条连续、可信、可追踪的时间现实。

这不是一个“给模型多一个工具”的项目。
这是要给 OpenClaw 增加一层可以长期维持人格沉浸感的时间现实层。

## 2. 最高产品目标

Timeline 的最高目标不是：
- 把 `timeline_resolve` 做成可调用工具；
- 给维护者多一个 `status` / `repair` 页面；
- 让 agent 在显式 tool call 测试里跑通。

Timeline 的最高目标是：

> 让安装了 `timeline-plugin` 的 OpenClaw，在用户看来具备真人一样的时间感知、状态连续性与回忆能力，并让这种时间现实能够被聊天、自拍和未来其他技能共同消费。

换句话说，用户不该感知到“现在是在调一个 timeline 工具”。
用户应该感知到“这个角色真的有自己的生活轨迹，而且前后说得通”。

## 3. 用户体验承诺

当 Timeline 达到目标态时，至少应满足以下体验：

### 3.1 当前状态回答要自然

用户问：
- “你在干嘛？”
- “你现在在哪？”
- “你今天怎么样？”

OpenClaw 应自然回答，而不是暴露工具或返回结构化痕迹。

### 3.2 近期回忆要像真实回忆

用户问：
- “最近有什么有趣的事吗？”
- “你这几天都在忙什么？”

OpenClaw 应先检索已有事实，再对空白窗口做人格化补全，而不是直接说“没有记录”。

### 3.3 状态要具有连续性

如果五分钟前系统生成了“正在打球”的状态，五分钟后再次被问“你在干嘛”，系统应优先推理“还在打球”，除非有更强的新事实把这个状态结束掉。

### 3.4 下游 skill 必须共享同一现实

自拍类 skill、persona 类 skill、其他场景化 skill 在需要“当前状态 / 最近状态 / 某时状态”时，应消费 Timeline 的统一结果，而不是各自再编一套状态。

### 3.5 用户看到的是角色，不是调试系统

调试与 trace 信息应服务维护者，不能渗透到用户侧自然聊天体验里。

## 4. Timeline 在系统中的角色

Timeline 是：
- OpenClaw 的时间事实与人格化状态装配层；
- temporal query 的统一解析与决策入口；
- timeline canon 的唯一受控写入方；
- persona 与下游 skill 的时间现实供应方。

Timeline 不是：
- 一个只靠 prompt 的“记忆写作技巧”；
- 一个只负责返回 JSON 的工具演示；
- 一个替代 OpenClaw 全局 memory 系统的搜索引擎；
- 一个可以随意覆盖既有记忆的自由编辑器。

## 5. 真相层级

系统必须遵守下面的事实优先级：

1. 当前会话与近邻会话中的硬事实
2. 已落盘的 canonical timeline memory
3. 长期语义记忆与 persona 文件中的弱事实信号
4. 基于真实世界时间、节日、周末、时间段的人格化生成推理

这意味着：
- 生成永远不能推翻更强事实；
- memory_search 不能否定会话硬锚；
- persona 可以塑造“像什么样的人会做什么”，但不能任意改写已经发生过的 canon。

## 6. 目标态系统能力

要实现北极星目标，Timeline 必须具备下面六类能力。

### 6.1 自然对话路由能力

系统必须能在自然问法中稳定识别 temporal intent，例如：
- 当下状态询问
- 最近回忆询问
- 显式过去时间询问
- 需要持续性推理的追问

这层能力的成功标准不是“用户显式要求调用 tool”，而是 agent 在正常聊天里自动走 Timeline。

### 6.2 确定性检索能力

系统必须按固定顺序读取：
- `sessions_history`
- `memory_get`
- `memory_search`
- `SOUL` / `MEMORY` / `IDENTITY`

并明确“当前问题到底在找哪一段事实”，而不是只要当天有内容就默认复用最后一条。

### 6.3 连续性推理能力

系统必须显式支持：
- 事件持续时长推理
- 状态转移推理
- 同日活动延续推理
- “已有旧事实但当前窗口缺事实”的 gap-fill 推理

这部分是 Timeline 走向“像真人”而不是“像日志查询器”的关键。

### 6.4 人格化生成能力

当没有既定事实时，系统必须基于：
- persona 设定
- 长期记忆
- 最近会话
- 当前时间
- 星期 / 节日 / 周末 / 日内时段

生成一个符合现实逻辑、真假难辨、但不与强事实冲突的状态或回忆。

这部分应以 LLM 为核心语义引擎，runtime 负责约束、结构化和写盘。

### 6.5 统一 canon 写入能力

所有由 Timeline 认定成立的新事实，都应通过 Timeline 自己的受控路径写入 canonical memory。

写入必须满足：
- append-only
- 幂等
- 单写者原则
- 路径安全
- 冲突可诊断

### 6.6 稳定消费能力

Timeline 输出必须让聊天层和下游 skill 都能稳定消费。

至少要能稳定提供：
- 当前/目标时间窗口
- 地点
- 行为
- 情绪
- 外观 / 着装
- 可信度
- 事实来源与生成性质

## 7. 最高验收标准

Timeline 是否“做成了”，不以工具注册是否成功为判断标准，而以以下体验标准为准。

### 7.1 聊天验收

在真实 OpenClaw 聊天里：
- “你在干嘛” 能自动承接到 Timeline；
- “最近有什么有趣的事吗” 能自动走近期回忆；
- 回复自然，不暴露工具调用措辞；
- 多轮后仍能保持时间连续性。

### 7.2 记忆验收

在无事实时：
- 系统能人格化生成非空结果；
- 生成结果可写入 canon；
- 追问时能复用或延续该 canon，而不是重新编。

### 7.3 连续性验收

对于短时间内的重复追问：
- 如果活动在常识上具有持续性，应优先视为仍在继续；
- 如果已经跨越合理结束窗口，应允许转入后继状态；
- 若有更强事实，应及时覆盖推理链。

### 7.4 下游联动验收

自拍类 skill 取用 Timeline 后：
- 时间、地点、事件、外观、状态要互相一致；
- 不允许聊天说“在家写东西”，自拍却生成“在海边散步”的割裂场景。

## 8. 当前实现与北极星目标的差距

当前仓库已经完成的部分：
- Timeline 插件与工具骨架已成立；
- `timeline_resolve` 已成为统一 runtime 入口；
- 受控 append-only 写入、trace、status、repair 已有基础实现；
- 基础 skill 路由与 OpenClaw runtime 集成已打通。

当前仍未完成的关键部分：
- 自然对话路由还未成为主验收标准；
- 检索逻辑仍偏向“当天最后一条”，不是“按问题命中目标事实”；
- gap-fill generation 与持续性推理还未成为显式能力；
- LLM generation seam 还没真正接成产品能力；
- 下游 skill 的稳定消费协议尚未定稿；
- 真实 E2E 验收仍偏 tool-centric，而不是体验-centric。

## 9. 设计原则

后续所有设计与实现，都应遵守以下原则。

### 9.1 体验优先于工具感

是否成功，先看用户体验是否自然，再看内部工具是否优雅。

### 9.2 事实优先于人格化

人格化生成只能补空，不得改史。

### 9.3 连续性优先于一次性回答

Timeline 不是一次性答题器，而是跨轮次维持同一现实。

### 9.4 LLM 负责高语义推理，runtime 负责秩序

不要试图用不断堆砌 heuristics 的方式替代真正的人格化生成。

### 9.5 下游消费必须稳定

Timeline 不只服务聊天，也必须服务自拍和未来技能。

### 9.6 trace 服务维护者，不能污染用户体验

调试能力必须更强，但用户侧话术必须更自然。

## 10. 后续文档与实现的约束

从本文件生效起，Timeline 项目的设计、实现与测试应遵守以下约束：

- 所有新设计讨论，先对齐本文件，再讨论局部接口。
- 所有新测试，应优先覆盖自然对话与下游联动场景，而不是只覆盖显式 tool call。
- 所有新能力，应说明它如何增强“真人时间感知与回忆体验”。
- 所有影响 persona / selfie / memory 的改动，应显式说明与 Timeline canon 的关系。

## 11. 一句话总纲

> Timeline 的最终形态，不是一个“能查时间线的工具”，而是 OpenClaw 的时间现实层。
