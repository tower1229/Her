# Timeline 集成验收清单

> 状态：当前有效
> 目的：作为 Timeline 后续重构与联调的统一验收口径
> 关联：`docs/timeline-north-star.md`、`docs/timeline-roadmap.md`

## 1. 使用原则

这份清单不再以“显式 tool call 是否成功”作为主要验收标准。

真正的验收口径是：

- 用户是否可以通过自然聊天触发 Timeline
- Timeline 是否先检索既定事实，再决定是否生成
- 生成结果是否具备连续性与 persona 一致性
- 聊天与下游 skill 是否共享同一条时间现实

## 2. 分层

### L0：运行时秩序层

验证范围：
- 读取顺序
- 写盘约束
- 冲突处理
- trace 与状态记录

说明：
这一层可以继续自动化，但它只能证明 runtime 秩序成立，不能证明用户体验成立。

### L1：resolver 决策层

验证范围：
- 当前窗口事实命中
- 窗口缺口识别
- 连续性推理
- 生成是否发生在正确时机

说明：
这一层是本轮重构的核心。

### L2：Agent 自然对话层

验证范围：
- 自然问法是否自动路由到 Timeline
- 回复是否自然
- 是否暴露工具感

说明：
这一层应优先成为主回归集。

### L3：下游 skill 联动层

验证范围：
- 自拍等 skill 是否消费 Timeline 的当前状态
- 聊天与产物是否一致

## 3. 核心验收目标

### A. 自然触发

当用户说：
- “你在干嘛”
- “你现在在哪”
- “最近有什么有趣的事吗”
- “你这几天都在忙什么”

Agent 应自动进入 Timeline 路径，而不是要求用户显式说“请调用 timeline_resolve”。

### B. 事实优先

如果会话或 canon 中已有明确事实：
- 必须优先复用
- 不得被 persona 文本或生成逻辑覆盖

### C. 生成补空

如果没有足够事实：
- 应允许人格化生成
- 不能直接返回“没有记录”
- 不能让 skill 自己编造一套与 Timeline 无关的状态

### D. 连续性

如果刚刚已经生成或命中过一个持续性活动：
- 短时间追问应优先视为仍在继续
- 不应每次都重新发明一个新活动

### E. 统一现实

如果聊天中当前状态是：
- 在家写东西
- 在咖啡馆休息
- 在球场打球

那么下游自拍或其他 skill 读取当前状态时，必须与之保持一致。

## 4. 关键场景

### 场景 1：当前状态命中既有事实

前置：
- 当天 canon 已存在一条与当前窗口相关的有效事实

用户：
- “你在干嘛”

期望：
- 自动走 Timeline
- 返回自然语言回答
- 不重复生成新事实
- 不重复写盘

### 场景 2：当前状态无事实，触发人格化生成

前置：
- 当前窗口没有足够事实
- persona / memory / 时间上下文可用

用户：
- “你在干嘛”

期望：
- 自动走 Timeline
- 生成一个符合 persona 和真实时间条件的状态
- 结果可写入 canon
- 回答自然

### 场景 3：连续性追问

前置：
- 五分钟前系统已生成“正在打球”或“正在咖啡馆坐着”

用户：
- “你现在还在干嘛”

期望：
- 优先推理为活动仍在持续
- 不重新换成一个无关状态
- 若状态已被新事实中断，则以新事实为准

### 场景 4：近期回忆补全

前置：
- 最近三天部分有 canon，部分没有

用户：
- “最近有什么有趣的事吗”

期望：
- 先复用已有事实
- 再对空缺部分进行合理补全
- 输出整体上像回忆，而不是像日志拼接

### 场景 5：下游自拍联动

前置：
- Timeline 已解析出当前状态

用户：
- “发张你现在的自拍”

期望：
- 自拍 skill 消费 Timeline 结果
- 地点、事件、外观、状态与聊天描述一致

## 5. 禁止通过但实际失败的情况

以下情况即使底层工具可调用，也应视为验收失败：

- 用户自然发问时没有自动路由到 Timeline
- Timeline 只会复用当天最后一条，不能按问题找事实
- Timeline 把应由 LLM 完成的生成工作 fallback 给脚本 heuristics
- 近期回忆只会返回空壳或“没有记录”
- 短时间追问时频繁跳变状态
- 聊天与自拍等下游 skill 使用了不同现实
- 用户回复里出现明显工具措辞、调试措辞或结构化痕迹

## 6. 当前重构阶段最需要补的自动化验收

### 第一组

- “你在干嘛”
- “你现在在哪”
- “最近有什么有趣的事吗”

### 第二组

- “你刚才不是在打球吗，现在还在吗”
- “你今天都忙了什么”

### 第三组

- “发张你现在的自拍”

## 7. 一句话判断标准

> Timeline 是否达标，不看它能不能被显式调用，而看它是否真的让 OpenClaw 在聊天和下游技能里共享同一条连续、可信的时间现实。
